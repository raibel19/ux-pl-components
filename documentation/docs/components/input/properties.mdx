# Referencia de Propiedades

El componente `Input` es altamente configurable a través de las siguientes propiedades. Están agrupadas por funcionalidad para facilitar su consulta.

---

Estas son las propiedades de nivel superior que controlan el comportamiento general y la apariencia básica del `Input`.`

:::info Para tu información

Cuando veas la palabra `"undefined"` como un valor dentro de un objeto de ejemplo, significa que el valor de esa propiedad es realmente `undefined`. Se muestra explícitamente para una mayor claridad de la estructura del objeto.

:::

## Propiedades de `ref` (`InputForwardRefType`)

Cuando usas una `ref` con el componente `Input`, el objeto `ref.current` tendrá acceso a los siguientes métodos para un control imperativo del componente.

| Método                         | Tipo                       | Descripción y Uso                                                                                                                                                                                                                                                                                                                                                   |
| :----------------------------- | :------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **`focus()`**                  | `() => void`               | <p className='max-w-3xl'>Pone el foco en el elemento `<input>`.</p>                                                                                                                                                                                                                                                                                                 |
| **`blur()`**                   | `() => void`               | <p className='max-w-3xl'>Quita el foco del elemento `<input>`.</p>                                                                                                                                                                                                                                                                                                  |
| **`get()`**                    | `() => HTMLElement │ null` | <p className='max-w-3xl'>Retorna el nodo del DOM del elemento `<input>` real para manipulación directa si es necesario.</p>                                                                                                                                                                                                                                         |
| **`subscribeFocus(callback)`** | `(callback) => void`       | <p className='max-w-3xl'>Permite suscribirse a los cambios de estado de `focus` del input. Es ideal para reaccionar cuando el usuario interactúa con el campo.</p><p className='mt-2'>**Parámetro:**<br/>`callback: (isFocus: boolean) => void`</p><p className='mt-2'>**Ejemplo en una línea:**<br/>`miRef.current?.subscribeFocus(foco => console.log(foco))`</p> |

---

### ¿ Como usar las propiedades `ref` ?

Este ejemplo muestra cómo declarar una `ref`, acceder a sus métodos dentro de un `useEffect` para poner el foco al montar el componente y suscribirse a los cambios de foco.

```jsx
import { useRef, useEffect } from 'react';
import { Input } from 'ux-pl-components/components/Input'; // Asegúrate que la ruta sea correcta
import { InputForwardRefType } from 'ux-pl-components/components/Input/interfaces/input'; // Asegúrate que la ruta sea correcta

function MiComponenteConRef() {
  const inputRef = useRef < InputForwardRefType > null;

  useEffect(() => {
    // 1. Poner el foco en el input cuando el componente se monta
    inputRef.current?.focus();

    // 2. Suscribirse a los cambios de foco para realizar acciones
    inputRef.current?.subscribeFocus((isFocused) => {
      console.log('El estado del foco del input es:', isFocused);
      // Aquí podrías, por ejemplo, mandar a llamar una API si el usuario deja el campo.
    });
  }, []);

  return (
    <div>
      <p>Este es un ejemplo de cómo usar la ref con el componente Input.</p>
      <Input ref={inputRef} textLabel="Input con Ref" />
    </div>
  );
}
```

##### Detalle de `subscribeBetween`

Esta función te permite reaccionar y recibir información sobre si el valor ingresado cumple con la validación de rango `between`.

- **Parámetros del Callback:** La función recibe un objeto con las siguientes propiedades booleanas:
  - `isLess`: `true` si el valor es menor al `min` del rango.
  - `isGreater`: `true` si el valor es mayor al `max` del rango.
  - `inRange`: `true` si el valor está dentro del rango `min` y `max`.
- **Ejemplo de Uso:**
  ```jsx
  <Input
    validations={{
      number: {
        between: {
          min: 5,
          max: 15,
          subscribeBetween: ({ isLess, isGreater, inRange }) => {
            console.log('Validación de rango:', { isLess, isGreater, inRange });
            if (!inRange) {
              // Lógica personalizada si el valor está fuera de rango
            }
          },
        },
      },
    }}
  />
  ```

## Sanitización de Entrada (`sanitize`)

La prop `sanitize` es un objeto que agrupa configuraciones para limpiar o "sanitizar" los valores ingresados, especialmente útil para inputs de tipo numérico.

<div className="admonition admonition-danger">
  <strong>Nota Importante:</strong> Toda la funcionalidad de <code>sanitize</code> solamente tiene efecto en inputs de
  tipo numérico.
</div>

### Detalle de `sanitize`

| Propiedad              | Tipo     | Default | Descripción                                                                                                                                     |
| :--------------------- | :------- | :------ | :---------------------------------------------------------------------------------------------------------------------------------------------- |
| **`maxDecimalDigits`** | `number` | `2`     | Número máximo de decimales permitidos en todas las fases de sanitización.                                                                       |
| **`onPropertyEntry`**  | `object` | `{...}` | Configuración de sanitización para los valores de entrada (`value` y `defaultValue`). [**Ver detalles.**](#detalle-de-sanitizeonpropertyentry)  |
| **`onInput`**          | `object` | `{...}` | Configuración de sanitización que se aplica mientras el usuario escribe caracter por caracter. [**Ver detalles.**](#detalle-de-sanitizeoninput) |
| **`onChange`**         | `object` | `{...}` | Configuración de sanitización que se aplica justo antes de que se emita el evento `onChange`. [**Ver detalles.**](#detalle-de-sanitizeonchange) |

---

#### Detalle de `sanitize.onPropertyEntry`

Define cómo se sanitizan los valores que llegan al componente a través de las props `value` y `defaultValue`. Es crucial para manejar la interacción con inputs controlados en React.

<details>
<summary>Explicación detallada del comportamiento con `useState`</summary>

Cuando usas `useState` para controlar el `value` de un `Input`, la configuración de `onPropertyEntry` interactúa directamente con los eventos `onChange` y `onInput`.

Por ejemplo, si `onPropertyEntry` tiene `removeDotIfLastCharacter: true` y tu `onChange` intenta actualizar el estado con un valor como `"123."`, cuando ese nuevo valor del estado se vuelve a pasar al `Input`, `onPropertyEntry` lo sanitizará a `"123"`. Esto puede impedir que el usuario escriba decimales si las configuraciones no están alineadas. La clave es entender que `onPropertyEntry` limpia el valor que "entra" al componente, afectando lo que el usuario ve finalmente.

</details>

**Propiedades del objeto `onPropertyEntry`:**

| Propiedad                                 | Tipo      | Default | Descripción                                                           |
| :---------------------------------------- | :-------- | :------ | :-------------------------------------------------------------------- |
| **`removeDecimalPoint`**                  | `boolean` | `false` | Si es `true`, elimina cualquier punto decimal del valor.              |
| **`removeNegativeSign`**                  | `boolean` | `false` | Si es `true`, elimina cualquier símbolo negativo (`-`).               |
| **`removeDotIfLastCharacter`**            | `boolean` | `true`  | Si el último carácter es un punto, lo elimina.                        |
| **`removeDecimalPointIfSingleCharacter`** | `boolean` | `true`  | Si el valor es solo un punto (`.`), lo convierte en una cadena vacía. |
| **`removeNegativeIfSingleCharacter`**     | `boolean` | `true`  | Si el valor es solo un guion (`-`), lo convierte en una cadena vacía. |

---

#### Detalle de `sanitize.onInput`

Define la sanitización que ocurre en tiempo real, cada vez que el usuario presiona una tecla. Está optimizado para permitir una experiencia de escritura fluida.

Por defecto, en esta configuración, las props como `removeDotIfLastCharacter` o `removeNegativeIfSingleCharacter` están en `false` para permitir que el usuario pueda escribir números negativos (`-25`) o decimales (`123.`) sin que el sanitizador elimine el `-` o el `.` al instante.

**Propiedades del objeto `onInput`:**
_(Este objeto comparte las mismas propiedades que `onPropertyEntry` pero con diferentes valores por defecto para una mejor experiencia de usuario al escribir)._

| Propiedad                                 | Tipo      | Default | Descripción                                                           |
| :---------------------------------------- | :-------- | :------ | :-------------------------------------------------------------------- |
| **`removeDecimalPoint`**                  | `boolean` | `false` | Si es `true`, elimina cualquier punto decimal.                        |
| **`removeNegativeSign`**                  | `boolean` | `false` | Si es `true`, elimina cualquier símbolo negativo (`-`).               |
| **`removeDotIfLastCharacter`**            | `boolean` | `false` | Si el último carácter es un punto, lo elimina.                        |
| **`removeDecimalPointIfSingleCharacter`** | `boolean` | `false` | Si el valor es solo un punto (`.`), lo convierte en una cadena vacía. |
| **`removeNegativeIfSingleCharacter`**     | `boolean` | `false` | Si el valor es solo un guion (`-`), lo convierte en una cadena vacía. |

---

#### Detalle de `sanitize.onChange`

Define la sanitización que se aplica al valor justo antes de que el componente emita el evento `onChange`. Esto es útil para limpiar el valor final antes de que llegue a tu lógica de estado.

Por defecto, esta configuración es más estricta que `onInput`. Por ejemplo, si el usuario deja `"123."` en el campo, `onChange` lo limpiará a `"123"` antes de enviarlo, ya que `removeDotIfLastCharacter` es `true`.

**Propiedades del objeto `onChange`:**
_(Este objeto comparte las mismas propiedades que `onPropertyEntry` y `onInput` pero con una combinación de valores por defecto pensada para el valor final)._

| Propiedad                                 | Tipo      | Default | Descripción                                                           |
| :---------------------------------------- | :-------- | :------ | :-------------------------------------------------------------------- |
| **`removeDecimalPoint`**                  | `boolean` | `false` | Si es `true`, elimina cualquier punto decimal.                        |
| **`removeNegativeSign`**                  | `boolean` | `false` | Si es `true`, elimina cualquier símbolo negativo (`-`).               |
| **`removeDotIfLastCharacter`**            | `boolean` | `true`  | Si el último carácter es un punto, lo elimina.                        |
| **`removeDecimalPointIfSingleCharacter`** | `boolean` | `true`  | Si el valor es solo un punto (`.`), lo convierte en una cadena vacía. |
| **`removeNegativeIfSingleCharacter`**     | `boolean` | `true`  | Si el valor es solo un guion (`-`), lo convierte en una cadena vacía. |

## Formato de Números (`formatter`)

La prop `formatter` es un objeto que permite formatear visualmente el valor numérico del input cuando este pierde el foco. Se basa en la API `Intl.NumberFormat` del navegador.

<div className="admonition admonition-info">
  <strong>Nota:</strong> El formateo es puramente visual. Cuando el input recupera el foco, el valor vuelve a su estado
  original sin formato. El valor enviado en el evento <code>onChange</code> tampoco incluye el formato.
</div>

| Propiedad           | Tipo      | Default                  | Descripción                                                                                                                                                                                                                     |
| :------------------ | :-------- | :----------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **`locale`**        | `string`  | (El local del navegador) | El código de localización a usar para el formato (ej. `'en-US'`, `'es-MX'`).                                                                                                                                                    |
| **`options`**       | `object`  | `{...}`                  | Un objeto con las opciones de formato de `Intl.NumberFormat`. [Ver documentación de MDN para todas las opciones](https://developer.mozilla.org/es/docs/Web/JavaScript/Reference/Global_Objects/Intl/NumberFormat/NumberFormat). |
| **`roundDecimals`** | `boolean` | `true`                   | Si es `true`, redondea los decimales según las reglas de `minimumFractionDigits` y `maximumFractionDigits`.                                                                                                                     |

**Ejemplo del valor por defecto de `options`:**

```js
{
  currency: 'MXN', // (Obtenido del locale, ej. para 'es-MX')
  style: 'decimal',
  minimumFractionDigits: 2,
  maximumFractionDigits: 2
}
```
